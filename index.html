Mir Gallery — Documentation

A visualization workbench for the Mindgut / Mir project. Browse reference images, curate selections, and create artifacts collaboratively with Claude.

References Tab

119 categorized reference images spanning scientific notation, generative art, data visualization, topology, and more. Each image is tagged with 2-4 descriptors for filtering.

  • Search — type to filter by name, tag, or description
  • Tag filters — click a tag pill to show only matching images
  • View modes — toggle between grid (compact) and large (detailed)
  • Lightbox — click any card to view fullscreen

Selection Mode

Click ☐ select in the toolbar to enter selection mode. In this mode, clicking cards toggles their selection instead of opening the lightbox. Selected cards show a blue border and checkmark.

  • A floating bar appears at the bottom showing the count
  • Click Open Workspace → to enter the workspace with your selection
  • Click clear to deselect all
  • Selections persist across page reloads (localStorage)

Workspace

The workspace is a dedicated view for discussing and planning artifacts. It shows your selected references as a filmstrip, with space for notes and artifact registration.

  • Filmstrip — horizontal scroll of selected references. Hover to reveal the × button to remove individual items.
  • Notes — free-form textarea for observations, themes, and ideas. Auto-saved to localStorage.
  • Export Brief — downloads a markdown file (or copies to clipboard) containing your selected references, notes, and suggested tags. Designed to be handed to Claude Code as context for building an artifact.
  • Register Artifact — once you've built a visualization (as an HTML file in artifacts/), register it here with title, description, tags, and status. The artifact is linked to its source references.

Artifacts Tab

Registered visualizations and prototypes. Each artifact card shows its status (draft/wip/stable), linked reference thumbnails, and opens in a new tab when clicked.

  • Artifacts created via the workspace are stored in localStorage and appear immediately
  • To make them permanent, copy the generated code snippet into the ARTIFACTS array in index.html
  • Reference cards that are linked to artifacts show a linked badge

Workflow: Reference → Artifact

  • 1. Browse references, find inspiration
  • 2. Enter select mode, choose your references
  • 3. Open workspace, write notes about what you want to build
  • 4. Export brief → hand to Claude Code
  • 5. Claude builds the artifact HTML
  • 6. Place file in artifacts/, register in workspace
  • 7. Iterate — create v2, v3, etc.
